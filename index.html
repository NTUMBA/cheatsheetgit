GIT est un logiciel de gestion de versions en mode distribué

Il s'agit d'un outil très pratique. En effet, il est très facile à apprendre et incontournable aussi bien pour de grands projets que pour ceux à ambition plus modeste. Il est bien plus performant que les outils SCM comme Subversion eu égard aux fonctions permettant des ramifications.

Dans tout processus de développement, la ligne de commande, à un moment donné, s'avère indispensable. Il n'est pas évident de mémoriser toutes ces commandes même les plus importantes, c'est pourquoi on peut, en cas de besoin, recourir à ce document structuré de sorte qu'il soit lisible, compréhensible et confortable à consulter.

QUELQUES CONSEILS UTILES POUR UNE MAITRISE EFFICIENTE

Pour bien commencer, les commandes à retenir:

git init

git clone ssh (ou http) ...

git add -A

git commit -m "First commit ou updates ..."

git status

git branch

git push origin nom_de_la branche

git pull [options] récupère et fusionne automatiquement une branche distante dans votre branche locale

Les commits font état des modifications

Il est recommandé de les rédiger fréquemment

Eviter de commit des tâches non abouties

Tester les codes avant de commit les actions

Les messages liés aux commits doivent être concis, précis et pertinents

Les versions de contrôle ne doivent pas être considérées comme étant un système de sauvegarde

S'accorder (en équipe) sur le flux des opérations: A (diapason); mieux, adopter une des méthodes agile (ex.: SCRUM) pour éviter quelque surprise ...-:)
